Virtualization Products

Overview

What Is Virtualization?

Virtualization products allow sharing the resources of one physical computer with multiple virtual machines. Each virtual machine resembles a physical computer but consumes only a fraction of the physical computer's resources. The resources used by the virtual machine are provided by the host computer. Virtualization allows you to run multiple different products and operating systems on the same hardware. For instance, the host computer may run Windows Server 2003 while virtual machines (also known as guests) may run Linux, Windows XP, and other operating systems, without any compatibility issues.

With virtualization, you can even overcome differences between processor architectures. For instance, can be installed on PowerPC-based Mac OS X computers, even though itself requires an Intel/AMD x86 processor.

supports all major virtualization products. You can install on an existing server computer by first installing a supported virtualization product and then installing into a virtual machine. This saves money and resources, and further hides the host hardware from , allowing you to use hardware and network configurations not directly supported by .

Other Virtualization Products

Such systems include Qemu and Xen in full virtualization mode. However, does not support installation into these alternative systems, and functionality cannot be guaranteed.

Installing Using Virtualization Products

Common Requirements

  • Virtual hard disk of at least 2 GB (about 1.86 GiB)
  • 256 MiB of memory for virtual machine (512 MiB recommended)
  • One or two virtual network interfaces (Ethernet)

Installing server on virtualization products is very similar to installing onto physical hardware. There are product specific variations on how virtual machines are created, started, and configured for server use (automatic starting on host startup, and so on). Please refer to your virtualization product manuals for more information. Instructions for VMware Server and KVM are described below as illustrations. Also see Common Post-Installation Steps below.

A prepackaged virtual appliance is available for VMware on the Download page. No product installation is necessary when using a virtual appliance: simply unpack the appliance files, review virtual machine configuration, and take the virtual machine into use.

Common Post-Installation Steps

After configuring the server, you should perform the following steps to ensure that the product performs as reliably and securely as possible.

  • Virtual machine boot order: Check that the virtual machine starts into when the virtual machine is rebooted. In particular, check that there is no virtual CD/DVD image in the virtual machine CD/DVD drive. Alternatively, change virtual machine boot order to prefer booting from hard disk instead of CD/DVD.
  • Virtual machine autostart on host reboot: If available in the virtualization product, ensure that the virtual machine starts automatically when the host reboots. This is important for instance with power outages, to ensure that remote access service is automatically restarted after the host computer reboot.
  • Take a backup or a snapshot of the virtual machine: Power off the virtual machine and take a backup of the virtual machine directory. Some virtualization products also provide a snapshot capability, which can be used instead. A backup or a snapshot allows you to roll back to an earlier configuration in case of a problem with the product (a configuration error, accidental deletion of user accounts, etc).
  • Host computer automatic reboot: Because most operating systems have trouble performing reliably over extended periods (months), it is recommended to set up an automatic reboot schedule for the host computer. For instance, the host computer could be configured to reboot once a week or once a month. This minimizes risk of host computer related remote access service disruptions. will automatically reboot the virtual machine periodically to minimize disruptions caused by the virtual machine.
  • Host computer network configuration: The host computer should not have a configured IP address in order to minimize risks of host computer intrusion. If the host computer needs to have an IP address (which is quite typical), host computer firewall settings should be carefully configured and the remote access should be reachable only through a firewall port forwarding function.
  • Host computer power management: Check host computer BIOS and operating system power management settings. The host computer should never power off, despite lack of any activity.
  • Host computer screensaver: Check host computer screensaver settings. The preferred screensaver is a simple black screen to minimize performance impact of the screensaver.
  • Host computer RAID settings: If your host computer uses RAID disk array, check that RAID settings are correct. In particular, check that the host computer alarms you somehow if integrity of the RAID configuration is threatened by a failing disk.
  • Host computer UPS settings: If your host computer is connected to a UPS, check that the UPS settings are correct. Check that UPS alarms are delivered to you using e.g. email, and simulate a power failure to ensure that the alarms are actually delivered.

Known Limitations

Virtual machines running should not be suspended. Suspend/resume causes a large time synchronization discrepancy which disrupts VPN service. Please shut down virtual machines instead of suspending them.

VMware (Workstation, Server, ESX Server)

VMware has several different virtualization products intended to a wide range of users; from desktop to data center use. They also provide free products, such as VMware Server and VMware Player. works best with VMware Server and VMware ESX Server. can be used with other VMware products as well, but they may be less ideal environments. For instance, VMware Workstation does not provide an easy and reliable way to start virtual machines automatically when the host computer boots up, which is important for a network service like .

To install using VMware Server, do as follows (these steps are for Linux version of VMware Server, but are very similar to the Windows version; ESX Server installation is slightly different but the same parameters apply):

  • Create virtual machine
    • Select File -> New -> Virtual Machine. Click Next.
    • Select Custom. Click Next.
    • Select Guest Operating System value 2. Linux, and select Ubuntu from the Version dropdown menu. Click Next.
    • Select an appropriate name for the virtual machine (e.g. Server), and check the storage location for virtual machine files. Click Next.
    • Select number of processors as One. supports multiprocessor environments, but in typical cases the extra performance is not necessary. Click Next.
    • In some VMware products you will be asked whether the new virtual machine should be made private (visible only to VMware user creating the virtual machine) or not. Select whichever is more appropriate for you. Click Next.
    • Specify memory size of 512 MB (recommended). 256 MB is the minimum supported by . Click Next.
    • Select Network Connection value Use bridged networking. This connects your virtual machine directly to the local Ethernet network. Click Next.
    • Select I/O Adapter Types value SCSI adapter / LSI Logic. In general the default selection is best. Click Next.
    • Select Create a new virtual disk. Click Next.
    • Select Virtual Disk Type value SCSI (Recommended). Click Next.
    • Specify size for new virtual disk as 2.0 (GB). Check the Allocate all disk space now and Split disk into 2GB files checkboxes. Click Next.
    • Select a name for the virtual disk file. The default is typically OK. Click Finish.
    • VMware will now create the virtual disk file, which should take about one minute. When complete, VMware will return you to the main administration window.
  • Finalize configuration and configure bootup from Live CD
    • Open virtual machine settings window by clicking Edit virtual machine settings.
    • Select Options tab.
      • Select Power and check the Power on after opening the virtual machine and Close after powering off or suspending checkboxes.
      • Select Snapshot and check the Disable snapshots checkbox.
      • Select Startup/Shutdown, select the Power on virtual machine under the label On host startup, and Shut down guest operating system under the label On host shutdown.
    • Select Hardware tab.
      • Select CD-ROM 1 (IDE 1:0) (or similar). Check the Connect at power on checkbox. Select the Use ISO image: option, and click the file browsing icon to locate the Live CD ISO image file.
      • Select Floppy 1 (or similar), and click Remove to remove the unnecessary floppy device.
      • Select Ethernet 1 (or similar). Ensure that Connect at power on checkbox is checked, and that the Bridged: Connected directly to to the physical network option is selected.
      • If you wish to use a configuration with two network interfaces (see Server Requirements), click Add.... Select Ethernet Adapter from the Hardware Types list and click Next. Select Bridged: Connected directly to the physical network option for Network Connection, ensure that the Connect at power on checkbox is checked, and click Finish.
    • Click OK to close the Virtual Machine Settings window.
  • Boot from Live CD and install server
    • Start the new virtual machine by clicking the green "play" icon (Power On).
    • Enter the BIOS setup menu of the virtual machine: get virtual machine focus by clicking on the virtual machine display area, and press F2 when instructed by the BIOS. The BIOS menu should appear.
    • Navigate to the Boot menu by pressing the right arrow key multiple times. Navigate the list of boot devices using the down arrow key until the CD-ROM Drive text is active. Press the plus key (on the keypad) until the CD-ROM Drive is first in the boot list.
    • Navigate to the Exit menu by pressing the right arrow key. Ensure that Exit Saving Changes is selected and press Enter. Confirm the save operation by selecting Yes and press Enter.
    • The Live CD should now boot in the virtual machine. The remaining installation steps are the same as in a normal installation. Select the virtual hard disk as the installation target. Remember to eject the CD-ROM or disable the CD drive of the virtual machine before initial configuration, so that boots correctly from the hard disk after installation. For more information, see Server Installation.
  • Finalize installation
    • Disable boot from the Live CD as follows. In the main VMware screen click Edit virtual machine settings. Select CD-ROM (IDE 1:0) (or similar), and clear the Connect at power on checkbox. Close the settings window by clicking OK.
    • Check the Common Post-Installation Steps above.

KVM

KVM (Kernel-based Virtual Machine) is a free, high quality virtualization technology built directly into modern Linux platforms. KVM provides high performance through hardware assisted virtualization. It thus requires a relatively modern PC with either Intel VT-x or AMD-V (also known as AMD Pacifica) processor virtualization technology for maximum performance. We suggest that you use a reliable and stable Linux distribution for deploying KVM. For instance, the Ubuntu Server is a good free platform for this purpose.

KVM management is command line based, which allows ordinary Linux scripts to be used for e.g. startup. An alternative solution is to use some GUI wrappers for KVM such as virt-manager. The following instructions assume that the host has X server configured. Installation without X server is similar, except that remote VNC connections are used throughout the installation.

To install on KVM, do as follows:

  • Login with proper permissions (e.g. as root). Read the KVM manual pages for an overview of KVM and its options. 
    $ man kvm
$ man kvm-qemu   # may vary depending on distribution
  • Create a virtual machine
    • Create a directory for the virtual machine: 
      $ mkdir /<path>/kvm/vpnease
$ cd /<path>/kvm/vpnease
    • Create a virtual disk for the virtual machine: 
      # in the virtual machine directory
$ dd if=/dev/zero of=disk.bin bs=1M count=4096  # 4 GiB
  • VPNease installation
    • If necessary in your distribution, start KVM kernel modules: 
      $ modprobe kvm        # for both AMD and Intel
$ modprobe kvm-amd    # for AMD
$ modprobe kvm-intel  # for Intel
    • Start the installer: 
      $ kvm -hda disk.bin \
      -cdrom vpnease_<version>.iso \
      -boot d \
      -std-vga \
      -net nic,vlan=0,model=rtl8139 \
      -net tap,vlan=0,ifname=vpnease0
    • You should configure network connectivity for the virtual machine at this point. The virtual machine will have a virtual eth0 Ethernet device, which will send and receive packets from the host vpnease0 (or whatever name was given above) TAP device.
      • The simplest approach is to configure a static address from a virtual subnet for the virtual machine, and assign the host TAP device an address from the same virtual subnet. Then, allow routing with NAT (MASQUERADE or SNAT) from the TAP device to the external network. Finally, add port forwarding for UDP ports 500 and 4500 (IPsec) and TCP ports 80 and 443 (web UI) from the external network to the virtual machine.
      • Several other configurations are possible. For instance, KVM supports a bridged connection mode. See KVM documentation for more information on network configuration alternatives. Note that the KVM "user" networking mode is not recommended because of its limitations.
    • Install normally, see Server Installation. Once installation is complete, power off the virtual machine.
    • To perform initial configuration, start KVM again with the following options (the difference to the previous options is that booting happens from the hard disk rather than the CD drive): 
      $ kvm -hda disk.bin \
      -boot c \
      -std-vga \
      -net nic,vlan=0,model=rtl8139 \
      -net tap,vlan=0,ifname=vpnease0
    • When initial configuration is done, ensure that the virtual machine gets a demo license. This indicates that networking functions at the basic level. You can also test client connections at this point. Once done, shutdown the product from the web UI and exit KVM.
  • Configure automatic startup for the server:
    • The KVM command line above is suitable for starting up the KVM virtual machine from e.g. /etc/rcN.d/ startup scripts, except that a different graphics device needs to be used. The following command line starts up the KVM virtual machine as above, but with VNC graphics suitable for startup from system scripts even without X being installed on the host (you can change the VNC display number to whatever you wish to use): 
      $ kvm -hda disk.bin \
      -boot c \
      -std-vga \
      -vnc :1 \
      -net nic,vlan=0,model=rtl8139 \
      -net tap,vlan=0,ifname=vpnease0
    • You should place a startup script in your system startup (consult your Linux distribution for specifics), which (1) probes the KVM kernel modules if necessary (see above); (2) starts up the KVM virtual machine e.g. as above; and (3) sets up the host networking correctly for the virtual machine. For the last step, you can use KVM interface scripts, or simply write your custom scripts.
  • Test your setup
    • Reboot the host computer.
    • Once the system is up and running, launch a VNC viewer to connect to the virtual machine graphics display: 
      # this varies per distribution
$ vncviewer :1.0
$ tightvncviewer :1.0
$ xvncviewer :1.0
$ xtightvncviewer :1.0
    • If everything has been set up correctly, you will see the web UI. Ensure that server gets an IP address, and that connection to the license server is up: this indicates that networking functions correctly.

QEMU is a virtualization platform almost identical to KVM, but it does not require hardware virtualization support (Intel VT-x or AMD Pacifica). However, it is considerably slower and thus not recommended.
