Using the VPN connection

You can open VPN connection from top right hand corner VPN menu:

  • Select Connect from the VPN menu.
  • If you have not save your password, a password dialog opens.
  • Enter your password and accept by clicking OK.
  • Ensure that connectivity to your company network resources works correctly by e.g. using your e-mail client or opening a web page from an intranet server.

You can disconnect the VPN connection by selecting Disconnect from the top right hand corner VPN menu.

VPN on Demand

Mac OS X 10.5 (Leopard) supports VPN on Demand, which automatically dials up the VPN connection when you access a network resource in your company network. To enable VPN on Demand, do as follows:

  • Open System Preferences and click Network. Select the VPN service (VPN (L2TP)) from the service menu on the left. Click Advanced and select the VPN on Demand tab.
  • Enter all domain names which need this VPN connection to be accessed, i.e. all relevant intranet services. For each domain name, click + (plus sign), enter the domain name into the Domain field, and select the name of your VPN connection profile from the Configuration dropdown. Each domain name is interpreted as a "wildcard" suffix; for instance, if you enter intra.example.com, all sub-domains are automatically included (e.g. server.intra.example.com). Click OK when you have finished adding all intranet addresses.
  • Click Apply and close the Network window. Open a browser window, and enter a URI to one of the intranet resources to the browser address bar. The VPN connection should now be automatically connected (you may be prompted for a password), and the intranet resource opened in your browser. The browser may confirm changes to your keychains, which you must allow.

If a configured domain name is accessible directly (without the VPN), the VPN is not dialed. This allows you to connect your computer to the intranet directly, without causing an unnecessary VPN connection to be opened.

Known limitations

  • When switching from one user to another, the VPN connection is automatically dropped by the operating system for security reasons. This behavior can be changed from VPN profile Advanced settings.
