Virtual Private Network (VPN) Overview

Virtual Private Network (VPN) products connect computers securely through insecure networks (such as the Internet), using encryption and other cryptographic techniques to secure data in transit. As a result, remote workers, customers, partners, and various company sites can be linked together securely, as if a secure "private network" existed between the parties. VPNs provide an extremely cost effective means of achieving secure connectivity, and have thus become popular in recent years.

VPN products typically provide two basic services:

  • Connecting remote workers and other individual users to a company network, which is often called "remote access VPN"
  • Connecting company or partner sites (networks) using a secure tunnel, which is often called "site-to-site VPN"

From a user perspective, remote access VPNs typically work like a modem connection: the remote access connection is opened, the user is authenticated using a username and a password, and a secure connection is created. User data traffic (in the form of IP data packets) can then be exchanged with the company network.

Site-to-site VPNs perform a similar function, but are autonomous. From user perspective they work similarly to a network router: data packets are sent to one site-to-site VPN device, which delivers them to the other site-to-site VPN device securely over the Internet, and the packets are then routed onwards. Many products provide one of these functions, while some products, like , provide both functions.

There are many VPN technologies in use, ranging from fully standard protocols (such as IPsec) to fully proprietary vendor-specific solutions (such as SSL VPNs). uses existing, reliable standards, and is based on the L2TP/IPsec VPN technology.

IPsec-based VPNs are the ideal technology for VPN connectivity, because they provide very high quality security (relied upon by banks and even military organizations), and they are packet based, which means that individual IP data packets are secured and transferred through the IPsec connection. This means that IPsec-based VPNs are fully transparent and are completely application independent. All the applications are protected automatically and can be used over the VPN connection. In particular, network services like Windows file sharing and Voice over IP (VoIP) are available without limitations.

Some other VPN technologies, like proprietary and SSL-based VPNs, are often application dependent. Many applications may not work without explicit support for the particular VPN solution.

Security and Standards

Like most VPN products, authenticates the remote user by username and password, and protects all user data sent between the client and the server computers. This effectively prevents attackers from gaining access to data in transit, and allows an employee to access company resources even from untrusted networks. A VPN solution should be deployed in conjunction with supporting security measures, such as client firewall, anti-virus and anti-spam scanning, and hard disk encryption. Together, VPN and other security measures allow flexible access to your data, while protecting against external threats.

protects user data by encrypting data exchanged between the client and the server computers. uses the IPsec security protocol to ensure the highest possible security for encrypted user data. IPsec is a standard developed by the Internet Engineering Task Force (IETF) and has been deployed commercially for almost 10 years. uses IPsec in conjunction with the L2TP protocol in a combination commonly referred to as L2TP/IPsec. See Encryption and Standards for details about cryptographic and compression algorithms.

Network Address Translation (NAT) is often a problem with many VPN solutions. supports the IPsec NAT traversal standard, allowing the product to work fully with all NAT and port forwarding devices. In particular, works correctly even when multiple client computers reside behind the same NAT device, a scenario which typically causes problems.
