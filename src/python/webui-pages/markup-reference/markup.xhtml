Page heading

Introduction

The markup used in the Nevow templates of the VPNease product is described and documented in full in this example markup file. This document provides the only "normative reference" for the conventions used.

The overall approach to the markup is as follows:

  • Existing HTML constructs are used exclusively; no "transformations" from own tags to HTML tags are performed (cf. transformation from Docbook to HTML).
  • CSS classes are used in all DIV/SPAN elements to provide context for CSS layout and styling. Whenever the meaning of the element is clear from the context (e.g. a P element inside a "section" DIV), class is omitted.
  • Ordinary sections (H1, H2, ...) are not used, because they use absolute nesting level and document fragments using such tags cannot thus be embedded in other fragments. Further, the heading elements don't nest, so there is no nesting information. Instead, DIV elements with class "section" are used; they have inner "heading" DIVs for labeling.
  • A side-effect of the heading approach is that numbering is not easily accomplished. It can be implemented as a post-transformation if necessary, because section DIVs nest properly.

This document contains a reference for the kind of XHTML markup that should be used in VPNease XHTML documents. The document covers the tags to be used, and the relevant CSS classes that go with them.

Basic text is a P elements without a class (the meaning is obvious from the DIV "section" context. Note that paragraphs are at the same level as the heading element. Paragraphs must not be nested, and must not contain lists (e.g. OL, UL) because XHTML forbids it.

Section headings are H1 elements; this makes them behave better in text browsers, though having all headings as "h1" is suboptimal.

Page-related elements

Most of the markup is not related to page structure at all (as much of the content is "included" onto a page). Pages have some styling that is not covered here (such as page layout through DIVs etc). Some elements are, though, and they are defined here.

Pages have a heading, which is a H1 with class "page-heading". It is uncertain whether this will remain in the final product: page heading may be fully programmatic and have special styling.

Section structure

Sections are DIVs with class "section". The DIV begins with a HR element with class "section" immediately following the DIV. The HR is followed by a H1 with class "heading", containing the section heading. After the H1, normal content continues. (The HR is used to make sections more readable in a text browser.)

Sections can be nested.

Second level heading

Sections are nested by placing a section DIV (directly) inside a section DIV.

Third level heading

Sections can be nested; paragraphs must never be nested. Further, lists must not occur inside paragraphs (XHTML forbids it).

Orderedlists (OL) and unordered lists (UL) do not need a class. Content for list items is placed in LI elements which also don't have a class. If multiple paragraphs of text are required for a list item, the LI element must contain multiple P tags and no direct content (direct child of LI).

Ordered list:

  1. Listitem #1
  2. Listitem #2
  3. Listitem #3

  4. Multiple paragraphs...

    ... inside a list item.

  5. Listitem #5

Unordered list:

  • Listitem #1
  • Listitem #2
  • Listitem #3

  • Multiple paragraphs...

    ... inside a list item.

  • Listitem #5

Styling text

Specific text styles are represented by SPAN or natural XHTML tags as follows:

  • Bold text: STRONG tag with "bold" class
  • Filename: STRONG tag with "filename" class
  • Registry path: STRONG with "registry" class
  • GUI label / system response: STRONG with "guilabel" class
  • User input: STRONG with "userinput" class
  • Keycap: STRONG with "keycap" class
  • FIXME note (inline): STRONG with "fixme" class

The STRONG tag is used to ensure that the "special text" is highlighted in a text browser. Text styles that don't need highlighting in a text browser should use the SPAN tag instead. However, no such styles are currently defined.

Preformatted text areas are represented by a PRE tag: 

File listing:
    - file 1
    - file 2
    - file 3

Links

Links are straightforward A tags with href and no class. They are styled based on context. Links can contain either text or an IMG tag.

  • Normal link with text: plain link. Link must not open a popup or go to an external site.
  • External link (opens a popup or goes to an external site): external link.
  • Image link: .

Anchors can be used normally inside a page by setting href=#foo in a link, and adding the corresponding A element with "name" attribute.
