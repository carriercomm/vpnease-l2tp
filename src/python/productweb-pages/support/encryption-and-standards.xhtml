Encryption and Standards

Overview

uses only the highest quality encryption standards to minimize risks to your data. The VPN standard used by is called L2TP/IPsec, which relies on IPsec encryption for protection of all VPN traffic, including user names, passwords, and actual data traffic. IPsec is a widely-deployed Internet standard for encryption and authentication of IP packets, and has been used commercially for almost 10 years. It uses the best available, up-to-date encryption algorithms, and provides military-grade encryption for end users.

In more detail, the L2TP/IPsec standard consists of three protocol layers. The lowest one, IPsec, provides all traffic security functions for all VPN data packets between the client and the server. The L2TP (Layer 2 Tunneling Protocol) runs on top of IPsec and provides a reliable transport mechanism for sending data units of the highest layer protocol, PPP (Point-to-Point Protocol). The IPsec layer protects all L2TP and PPP data, resulting in a highly secure VPN system, despite minimal security functions of L2TP and PPP.

Cryptographic and Compression Algorithms Used

The cryptographic algorithms used in IPsec encryption and authentication depend on the client platform. The client and server negotiate the most secure available algorithms during IPsec connection setup. Data compression is provided by the PPP protocol, with a similar automatic negotiation mechanism.

will automatically use the strongest encryption and best compression algorithm available for a particular VPN client, without any configuration.

IPsec employs one set of algorithms for connection negotiation (using the Internet Key Exchange, IKE, protocol) and another set for encryption of all data traffic, including user names and passwords (using the IPsec ESP protocol). Abbreviations and terms used in the table:

  • IKE encr: IKE (key exchange) encryption algorithm
  • IKE auth: IKE (key exchange) authentication and integrity protection algorithm (pre-shared key authentication used in all cases)
  • IKE D-H: IKE (key exchange) Diffie-Hellman group used to generate a random session key for IKE encryption algorithm
  • ESP encr: IPsec ESP (bulk data) encryption algorithm
  • ESP auth: IPsec ESP (bulk data) authentication and integrity protection algorithm
  • ESP D-H: Diffie-Hellman group used by IKE to generate a random session key for IPsec ESP (bulk data) encryption algorithm (if not used, IKE uses the same Diffie-Hellman computation used for IKE encryption)
  • Compression: PPP layer compression algorithm

Other platforms, such as Linux, will use one of the following algorithms:

  • IKE encryption: 3DES-CBC or AES-128-CBC
  • IKE authentication: MD5 or SHA1 (with pre-shared key)
  • IKE Diffie-Hellman: 1024-bit, 1536-bit, 2048-bit or larger MODP
  • ESP encryption: 3DES-CBC or AES-128-CBC
  • ESP authentication: HMAC-MD5-96 or HMAC-SHA1-96
  • ESP Diffie-Hellman: none, or 1024-bit, 1536-bit, 2048-bit or larger MODP
  • Compression: PPP Deflate or MPPC/MPPE (or none); IPcomp not supported
