Server Requirements

Server Hardware Requirements

  • Processor: 32-bit or 64-bit x86, recommended 2 GHz or more
  • Memory: minimum 256 MiB
  • Networking: one or two 10 Mbps or faster Ethernet network interfaces (NICs) depending on the network configuration
  • Storage: 2 GB or larger, dedicated storage device for installation. Storage device can be either hard drive (USB, IDE, SCSI, Serial ATA) or flash memory (USB stick, IDE Compact Flash)
  • BIOS: if USB stick installation is used, must be capable of booting from a USB HDD. If installation target is smaller than 4 GB, requires a BIOS capable of booting from hard disks larger than 512 MB.

Server Hardware Compatibility

server is based on the Ubuntu Linux distribution, with up-to-date hardware compatibility for all standard PC hardware. The latest is based on Ubuntu 6.06 LTS (Dapper Drake). For more information on hardware compatibility issues, see:

  • Ubuntu's support pages
  • Ubuntu's hardware support Wiki
  • Documentation for Ubuntu 6.06 LTS (Dapper Drake)

Although is based on the Ubuntu distribution, it is not supported by Ubuntu. Contact support or your service provider in all support matters. Please do not contact Ubuntu for support.

A few practical tips in choosing hardware:

  • Select recent but not the newest possible hardware to minimize risks with immature hardware
  • Use well-known, quality storage devices with long guarantee periods (3 years or more)
  • Flash-based storage devices can be considerably slower than magnetic storage devices, but have no moving parts and can provide more reliable long term operation
  • Favor low temperature and/or fanless components over more powerful, fan-cooled components
  • Check Linux (Ubuntu) compatibility from your seller
  • If Linux compatibility is uncertain, consider using a virtualization product as a platform for the product. This eliminates compatibility issues and allows you to choose any host operating system (e.g. Windows or other Linux variant)

Network Requirements

  • Internet connection speed: minimum 512 kbps.
  • Internet Service Provider (ISP) must not block the following protocols and ports:
    • UDP/500 (IPsec)
    • UDP/4500 (IPsec)
  • The following protocols and ports are optional for usage:
    • TCP/80 (HTTP, administrator and user web user interface access)
    • TCP/443 (HTTPS, administrator and user web user interface access)
    • IP protocol 50 (IPsec ESP without NAT traversal, required for clients without NAT traversal support)

Software Requirements

server has a web-based administrator interface, which requires Javascript support. server can be configured with the local Firefox browser automatically installed on the server. To use the administrator interface from another computer the following browsers are supported:

  • Firefox (1.5 or newer)
  • Internet Explorer (6.0 or newer)
  • Safari (2.0.1 or newer)

also has a separate web-based interface for VPN users, which provides user account management and guides for configuring the VPN client for supported operating systems. user web interface does not require Javascript and works with almost any browser, such as:

  • Firefox (1.5 or newer)
  • Internet Explorer (5.0 or newer)
  • Safari
  • Opera
  • Konqueror
  • W3m

Virtualization Products

In addition to using dedicated hardware, server can also be installed on existing hardware by using a virtualization product, such as VMware, Microsoft Virtual PC, Microsoft Virtual Server, Microsoft Hyper-V, KVM (free), or Parallels. See Virtualization Products for more details.

Installation on top of a virtualization product is recommended if:

  • There is existing server hardware that can comfortably share some capacity for terminating remote access connections. Typically only about 10% of server capacity is required to run a remote access gateway, yielding great hardware savings. If your company manages customer equipment, this is a great opportunity for providing additional options to your customer with minimal investment.
  • Your hardware is not compatible with the Linux distribution (Ubuntu) which is used as a basis for the product. Virtualization can overcome hardware compatibility issues, because any host operating system (e.g. Windows or other Linux variant) can be installed on the host hardware. will only see standardized virtual hardware.
  • Your networking hardware is not Ethernet based (e.g. PPPoE or a modem connection). The virtualization product will hide the actual networking hardware and present a virtual Ethernet device for .
  • You plan to install multiple remote access servers with different configurations, or you need some supporting servers and wish to minimize hardware costs.

has been tested with all major virtualization products. See Virtualization Products for more detailed information.

High Reliability Installations

For the high reliability installations we recommend one of the following:

  • Use a virtualization product and configure RAID hardware to the host computer. This provides storage reliability for the server as well.
  • Use a reliable USB stick (flash memory) or Compact Flash IDE (SATA) hard disk instead of a normal (magnetic) hard disk. Hard disks are a common cause of failure in server systems, and flash-memory-based hard disks avoid this common cause.

does not support RAID devices directly. To use software or hardware RAID, use a virtualization product and configure RAID in the host computer.

PC Hardware Reliability Tips

  • Minimize the number of cooling fans and invest in quality fans. Cooling fans are a common source of failure. A failing fan results in an increased computer temperature leading to increased risk of processor and other chip instability, increased risk of expansion card failures, and decreased lifetime of hard disks.
  • Check temperature. Allow the system to run for a day or two, and check case temperature. You can typically check the temperature values by restarting the computer and entering the BIOS setup screen. BIOSes typically display temperature and fan information in one of their information screens. Refer to your computer hardware manuals for information about expected operating temperatures.
  • Use quality components. Purchase quality components (motherboard, memory, network card). Low quality components may lead to failures which are difficult to trace and correct.
  • Run a "torture test". Run torture tests on your hardware to ensure the hardware performs correctly under heavy load. Common hardware torture tests include: Memtest86 (included on the Live CD), Prime95, Super PI, SiSoftware Sandra, and BOINC.
  • Replace hard disks before they fail. If you use a normal magnetic hard disk, replace it with a new one before it fails. Typical lifetime of a hard disk in normal use is 3-5 years. Check your hard disk data sheet for mean-time-to-failure (MTTF) estimates.
